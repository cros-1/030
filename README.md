# مشروع منصة قطاع عمليات وسط الرياض

محتويات الحزمة:
- index.html — الصفحة العامة
- admin.html — لوحة التحكم (تدعم Firebase Authentication)
- styles.css — أنماط الصفحة
- app.js — جافاسكربت الواجهة العامة (Firebase: قراءة الثيم، الإعلان، الروابط، لوحات التميز)
- admin.js — جافاسكربت لوحة التحكم (مصادقة، رفع صور، إضافة روابط، حفظ إعلان، تغيير الثيم)
- firebase.rules (نماذج لقواعد Firestore / Storage)
- README.md — هذا الملف

## خطوات سريعة للتشغيل
1. أنشئ مشروع Firebase (Firestore, Storage, Authentication).
2. استبدل القيم `REPLACE_ME` في ملفي app.js و admin.js بتكوين مشروعك من Firebase Console.
3. أنشئ مستخدم مشرف في Authentication (Email/Password) لاستخدام لوحة التحكم.
4. لتجربة كاملة ينصح باستخدام Firebase Hosting أو خادم محلي عبر http(s).

## قواعد أمان مقترحة (أمثلة)
- Firestore: السماح بالقراءة للعامة والكتابة للمصادقين فقط.
- Storage: السماح بالقراءة العام (إن أردت) والكتابة للمصادقين فقط.

ملف firebase.rules مرفق في الحزمة.

## أفكار تحسين (مفصّلة)
1. **دور المستخدم وصلاحيات متقدمة**
   - أنظمة أذونات: مشرف، محرر، مشاهد. السماح لمشرف بتحرير كل شيء، والمحرر بإضافة/تعديل لوحات، والمشاهد للقراءة فقط.
2. **معاينة وقتية (Live preview)**
   - عند تعديل الثيم أو الإعلان في لوحة التحكم تُعرض المعاينة مباشرة في إطار مصغّر دون الحاجة لإعادة تحميل الصفحة.
3. **تعليقات وتقييم للّوحات**
   - إضافة إمكانية للمستخدمين لتقييم لوحة التميز (نجوم) أو كتابة تعليقات/ملاحظات — مع قواعد للتحكم بالسبام.
4. **جدولة ونشر الإعلانات**
   - إمكانية جدولة بدء/انتهاء الإعلان بدل تفعيله فوراً. دعم تكرار وجدولة زمنية.
5. **إدارة ترتيب اللوحات**
   - السحب والإفلات (drag & drop) لترتيب ظهور لوحات التميز وتخزين ترتيب العرض في Firestore.
6. **معالجة وتحجيم الصور**
   - استخدام Cloud Functions لتحويل الصور لأحجام أصغر وتوليد WebP وتحسين الأداء.
7. **نسخ احتياطي وإصدار للنشر (Draft/Publish)**
   - دعم مسودات للّوحات والإعلانات ونظام تسجيل تغييرات (history) مع إمكانية التراجع.
8. **بحث وفلاتر للّوحات**
   - فلاتر بحسب القسم، التاريخ، أو أفضل التقييمات مع بحث نصي.
9. **تحليلات وقياس الأداء**
   - عداد للزيارات، تفاعل المستخدم، ومقاييس عرض اللّوحات باستخدام Google Analytics أو Firebase Analytics.
10. **إشعارات**
    - إشعارات Push للمشتركين عند إضافة لوحة جديدة أو تفعيل إعلان مهم.
11. **تحسين الواجهة (UX)**
    - تصميم واجهة أفضل للهواتف، تحسين السهولة، إضافة صور افتراضية عند غياب صور، وإظهار حالة التحميل.
12. **نسخة قابلة للطباعة / PDF**
    - توليد صفحة قابلة للطباعة أو تصدير لوحات إلى PDF للعرض الميداني.
13. **تحكم باللغات (i18n)**
    - دعم لغات متعددة (عربي/إنجليزي) باستخدام ملفات ترجمة.
14. **سياسات أمنية وRate Limiting**
    - حماية نقاط النهاية من محاولات إساءة الاستخدام، ومنع رفع ملفات كبيرة جداً.
15. **اختبار آلي وتكامل مستمر**
    - إعداد CI/CD لنشر التغييرات تلقائياً بعد الاختبارات.

## ملاحظات
- إن رغبت أعد لك نسخة مع Firebase Hosting مُهيّأة (index/admin) وقواعد Firestore جاهزة و `firebase.json` وملف deploy script.
- إن رغبت أقدّم واجهة محسّنة بتصميم مودرن (React/Vue) أو إضافة دعم للـSSR أخبرني وسأعدّ ذلك.
